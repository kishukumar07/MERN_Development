JavaScript Closures - Interview Notes

Definition

A closure is a function that retains access to its parent function's variables, even after the parent function has executed.

Key Concepts

  • Lexical Scoping: JavaScript uses lexical scoping, meaning a function's scope is defined by where it is written in the code.
  • Inner Function Access: Functions inside another function can access the outer function's variables.
  • Closure Creation: When an inner function is returned or referenced outside its scope, it forms a closure.

Why Use Closures?

  • Data Encapsulation: Helps create private variables and prevent unintended modifications.
  • Memory Efficiency: Stores only required variables instead of polluting global scope.
  • Function Factories: Can create multiple instances of functions with different values.
  • Event Handling: Useful in setting up event listeners while keeping access to specific data.
  • SetTimeout & SetInterval: Retains reference to variables when executing delayed functions.

Example

function outerFunction() {
    let outerVariable = "Hello";

    function innerFunction() {
        console.log(outerVariable); // Accessing the outer variable
    }

    return innerFunction;
}

let myClosure = outerFunction();
myClosure(); // Output: Hello

Key Takeaways

  • Closures allow functions to maintain access to their outer scope.
  • They help in data encapsulation and state persistence.
  • Widely used in JavaScript for event handling, timers, and functional programming.
